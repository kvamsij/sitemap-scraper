Project: "Sitemap Scraper"

Description:

    Develop a sitemap scraper that accepts a domain name as input, reads the `robots.txt` file, and extracts sitemap URLs. The scraper should handle both standard and `.gz` compressed sitemaps, filter product-specific sitemaps, and log events while writing the extracted data to a file in the desired format. The scraper should also filter URLs based on disallowed paths in `robots.txt` and identify product URLs using configurable patterns.

Setup:

1. Install the following dependencies:
   - `typescript` for type safety.
   - `jest` for unit testing.
   - `axios` for HTTP requests.
   - `ts-node` for running TypeScript files directly.
   - `xml2js` for parsing XML sitemaps.
   - `csv-writer` for writing CSV files.
   - `slugify` for generating slugified folder names.
   - `winston` for centralized logging.

2. Ensure the project is initialized with a `package.json` file and includes scripts for building, running, and testing the application.

Development Plan:

1. **Input Handling**:
   - Create a class (`DomainInput`) to accept and validate domain name input from the terminal.
   - Ensure the domain is properly formatted (e.g., adds `https://` if missing).

2. **Fetching `robots.txt`**:
   - Create a class (`RobotsFetcher`) to fetch the `robots.txt` file from the provided domain.
   - Use the `httpRequest` utility for making HTTP requests.

3. **Parsing `robots.txt`**:
   - Create a class (`RobotsParser`) to parse the `robots.txt` file to:
     - Extract allowed and disallowed paths.
     - Identify and extract sitemap URLs.

4. **Filtering Product Sitemaps**:
   - Create a class (`ProductSitemapFilter`) to filter and identify product-specific sitemaps from the extracted sitemap URLs.

5. **Fetching URLs from Sitemaps**:
   - Create a class (`SitemapUrlFetcher`) to fetch all URLs listed in both standard and `.gz` compressed sitemap files.
   - Handle nested sitemaps by recursively fetching and parsing them.
   - Use `xml2js` to parse XML sitemaps and handle namespaces.

6. **Filtering URLs**:
   - Implement a class (`UrlFilter`) to filter URLs based on:
     - Disallowed paths in `robots.txt`.
     - Configurable product URL patterns (e.g., `/product/`, `/item/`, `/sku/`, `/p/`, `/products/`, and query parameters like `?productId=`).

7. **Writing to File**:
   - Create a class (`FileWriter`) to write the extracted URLs to a file in the desired format (`csv`, `json`, or `txt`).
   - Organize the output into folders named after the slugified domain (e.g., `www-myprotein-com` for `https://www.myprotein.com`).

8. **Centralized Error Handling**:
   - Implement a centralized error-handling mechanism (`ErrorHandler`) to manage exceptions gracefully.
   - Exit the application for unknown errors after logging the details.

9. **Centralized HTTP Requests**:
   - Use the `httpRequest` utility for all HTTP requests, ensuring consistent error handling and logging.

10. **Testing**:
    - Write unit tests for each class and method using `jest`.
    - Mock external dependencies (e.g., `axios`) to isolate functionality during testing.

Development Workflow:

1. Initialize the project with TypeScript and required dependencies.
2. Set up a folder structure for better organization:
   ```
   src/
     ├── input/                # Handles domain name input
     ├── fetch/                # Fetches robots.txt and sitemaps
     ├── parse/                # Parses robots.txt
     ├── filter/               # Filters product-specific sitemaps and URLs
     ├── log/                  # Centralized logging system
     ├── write/                # Writes data to files
     ├── process/              # Processes sitemaps and URLs
     ├── tests/                # Unit tests
   ```
3. Write unit tests for each module in the `tests/` directory.
4. Implement functionality incrementally, ensuring all tests pass.
